## 2025-05-15 - [Security] JWT Signature Validation in Auth Middleware
**Vulnerability:** The authentication middleware was extracting JWT tokens from the Authorization header but failing to validate their signatures, allowing anyone to bypass authentication by providing a forged JWT with a validly formatted payload but no valid signature.
**Learning:** Middleware that only "attaches" tokens for downstream services to verify is inherently dangerous if it doesn't also perform an initial signature check. Downstream handlers (like `chat.ts`) might assume the presence of a token in the context implies it has already been verified, leading to potential authorization bypasses.
**Prevention:** Always perform a full cryptographic signature check at the edge/middleware level before populating trusted context variables like `userId`. Ensure the JWT secret is securely managed and accessed from the environment (e.g., `c.env.SUPABASE_JWT_SECRET`). Refactor common validation logic into a shared utility to prevent inconsistent security checks across different middleware functions.
