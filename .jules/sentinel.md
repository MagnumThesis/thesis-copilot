## 2024-05-18 - [Overly Permissive CORS with Credentials in Cloudflare Workers]
**Vulnerability:** The Hono `cors()` middleware in `src/worker/routes/auth-routes.ts` was configured with `origin: '*'` while simultaneously setting `credentials: true`. This combination allows any site to perform authenticated cross-origin requests, leading to potential Cross-Site Request Forgery (CSRF) or data leakage vulnerabilities.
**Learning:** In Cloudflare Workers running Hono, wildcard origins are particularly dangerous when combined with credentialed requests (like cookies or `Authorization` headers) on sensitive routes. The environment handles `origin: '*'` + `credentials: true` inconsistently across browsers, and it explicitly violates modern CORS security standards.
**Prevention:** Always implement a dynamic origin resolver function for CORS on sensitive or credentialed endpoints. Extract the allowed origin securely from the environment context (`c.env` or `import.meta.env`) and explicitly validate the incoming request's `origin` against allowed domains or specific development environments (e.g., `localhost`). Never use wildcard origins with `credentials: true`.
